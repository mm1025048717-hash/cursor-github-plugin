# GitHub AI 插件使用指南

## 🚀 快速开始

### 1. 安装插件

1. 打开 Cursor IDE
2. 按 `Ctrl+Shift+P` (Windows/Linux) 或 `Cmd+Shift+P` (Mac) 打开命令面板
3. 输入 "Extensions: Install from VSIX"
4. 选择编译好的 `.vsix` 文件安装

### 2. 配置 API 密钥

1. 打开设置 (`Ctrl+,` 或 `Cmd+,`)
2. 搜索 "GitHub AI"
3. 配置以下选项：
   - **GitHub Token** (可选): 提高 API 速率限制
     - 获取方式: GitHub → Settings → Developer settings → Personal access tokens → Generate new token
     - 权限: 只需 `public_repo` 权限
   - **DeepSeek API Key** (必需): 用于 AI 代码修改
     - 获取方式: 访问 [DeepSeek 官网](https://www.deepseek.com/) 注册并获取 API Key
   - **下载路径**: 项目下载的本地路径（默认: `${workspaceFolder}/github-projects`）

## 📖 功能使用

### 功能 1: 搜索并下载 GitHub 项目

1. 按 `Ctrl+Shift+P` 打开命令面板
2. 输入 "GitHub AI: 搜索项目"
3. 用自然语言描述你想要找的项目，例如：
   - "找一个 React 的待办事项应用"
   - "搜索 Python 的机器学习库"
   - "找 TypeScript 的项目管理工具"
4. 从搜索结果中选择一个项目
5. 项目会自动下载到配置的路径
6. 可以选择立即打开项目

**示例对话：**
```
用户: "找一个 React 的待办事项应用"
→ AI: 搜索 GitHub，找到多个匹配项目
→ 用户: 选择 "facebook/react-todo-app"
→ AI: 自动下载并提示是否打开
```

### 功能 2: 自然语言修改代码

1. 在 Cursor 中打开任意代码文件
2. 按 `Ctrl+Shift+P` 打开命令面板
3. 输入 "GitHub AI: 对话修改代码"
4. 描述你想要做的修改，例如：
   - "添加错误处理"
   - "优化这个函数的性能"
   - "添加详细的注释"
   - "重构这段代码使其更清晰"
5. AI 会分析代码并生成修改后的版本
6. 选择 "应用"、"预览" 或 "取消"

**示例对话：**
```
用户: 打开 app.js 文件
用户: 执行 "GitHub AI: 对话修改代码"
用户: "添加 try-catch 错误处理"
→ AI: 分析代码，生成带错误处理的版本
→ 用户: 选择 "应用"
→ AI: 代码已更新
```

### 功能 3: 管理已下载项目

1. 按 `Ctrl+Shift+P` 打开命令面板
2. 输入 "GitHub AI: 打开已下载项目"
3. 从列表中选择项目
4. 项目会在新窗口中打开

## 💡 使用技巧

### 搜索技巧
- **指定语言**: "找一个 Python 的 Web 框架"
- **描述功能**: "搜索用户认证库"
- **组合条件**: "找 TypeScript 的图表可视化库"

### 代码修改技巧
- **具体描述**: "在这个函数中添加参数验证"
- **代码优化**: "优化这个循环的性能"
- **添加功能**: "添加一个计算总数的函数"
- **代码重构**: "将这个类拆分成多个小类"

### 工作流建议
1. 先搜索并下载项目
2. 打开项目了解结构
3. 使用 AI 修改功能快速定制
4. 继续用自然语言对话完善代码

## ⚠️ 注意事项

1. **API 限制**:
   - GitHub API: 未认证 60 次/小时，认证后 5000 次/小时
   - DeepSeek API: 根据你的套餐限制

2. **代码安全**:
   - AI 生成的代码需要人工审查
   - 确保理解修改内容后再应用

3. **项目管理**:
   - 定期清理不需要的项目
   - 注意下载路径的磁盘空间

## 🐛 常见问题

**Q: 搜索失败，提示速率限制？**
A: 配置 GitHub Token 可以提高速率限制。

**Q: AI 修改代码失败？**
A: 检查 DeepSeek API Key 是否正确配置，账户是否有余额。

**Q: 下载的项目在哪里？**
A: 默认在 `${workspaceFolder}/github-projects`，可以在设置中修改。

**Q: 可以删除已下载的项目吗？**
A: 目前需要手动删除，后续版本会添加删除功能。
