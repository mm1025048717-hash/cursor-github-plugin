  ✅ 正确启动 GitHub AI 插件的步骤
═══════════════════════════════════════════════════════════

⚠️ 重要：如果看到"调试 Markdown"的提示，请点击"取消"！

📋 正确步骤：
─────────────────────────────────────────────────────────

1️⃣ 关闭所有打开的 Markdown 文件
   • 关闭 "快速开始.md" 等文档文件
   • 确保在文件浏览器中，而不是在编辑器中

2️⃣ 确保在 cursor-github-plugin 文件夹中
   • 左侧文件浏览器显示 cursor-github-plugin 为根目录
   • 可以看到 src/、out/、package.json 等文件

3️⃣ 选择正确的调试配置
   • 点击左侧调试图标（虫子图标）或按 Ctrl+Shift+D
   • 在顶部下拉菜单选择："Run Extension"
   • 不要选择其他配置

4️⃣ 按 F5 启动
   • 会打开新的 Cursor 窗口
   • 标题显示 [Extension Development Host]

5️⃣ 在新窗口中打开助手
   • 按 Ctrl+Shift+P
   • 输入：GitHub AI: 打开助手
   • 回车

─────────────────────────────────────────────────────────

💡 如果还是遇到问题：
─────────────────────────────────────────────────────────

方法 A：手动选择调试配置
1. 按 Ctrl+Shift+D 打开调试面板
2. 顶部下拉菜单选择 "Run Extension"
3. 点击绿色播放按钮

方法 B：使用命令面板
1. 按 Ctrl+Shift+P
2. 输入：Debug: Start Debugging
3. 选择 "Run Extension"

─────────────────────────────────────────────────────────

✅ 记住：要调试的是扩展，不是 Markdown 文件！
