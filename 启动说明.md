# 🚀 启动插件指南

## 方法 1: 在 Cursor 中直接运行（推荐）

### 步骤：

1. **打开项目文件夹**
   - 在 Cursor 中打开 `cursor-github-plugin` 文件夹
   - 确保这是工作区根目录

2. **启动调试**
   - 按 `F5` 键
   - 或点击菜单：运行 → 启动调试
   - 或点击左侧调试按钮（虫子图标）→ 点击绿色播放按钮

3. **等待新窗口**
   - 会打开一个新的 Cursor 窗口
   - 标题栏显示 `[Extension Development Host]`
   - 插件已自动加载

4. **配置 API Key**（首次使用）
   - 在新窗口中按 `Ctrl+,` 打开设置
   - 搜索 `githubAI.deepseekApiKey`
   - 输入你的 DeepSeek API Key

5. **打开助手**
   - 在新窗口中按 `Ctrl+Shift+P`
   - 输入：`GitHub AI: 打开助手`
   - 或输入：`GitHub AI`
   - 选择第一个命令

6. **开始使用**
   - 对话界面会打开
   - 直接用自然语言对话即可！

## 方法 2: 打包后安装

如果你想把插件打包成 .vsix 文件：

```bash
cd cursor-github-plugin
npm install
npm run compile
npm run package
```

然后在新窗口中：
1. 按 `Ctrl+Shift+P`
2. 输入：`Extensions: Install from VSIX...`
3. 选择生成的 `.vsix` 文件

## 🐛 常见问题

**Q: 按 F5 没反应？**
A: 
- 确保在 Cursor 中打开了 `cursor-github-plugin` 文件夹
- 检查 `.vscode/launch.json` 是否存在
- 尝试手动选择调试配置

**Q: 新窗口打开了但没有插件？**
A:
- 检查 `out/extension.js` 是否存在
- 查看输出面板（查看 → 输出 → 选择"扩展"）
- 检查是否有错误信息

**Q: 找不到命令？**
A:
- 确保插件已激活
- 在新窗口中查看输出面板的日志
- 尝试重新加载窗口（`Ctrl+R`）

## 📝 调试技巧

1. **查看日志**
   - 在原窗口（非 Extension Development Host）查看输出
   - 选择 "GitHub AI Plugin" 或 "扩展" 输出

2. **断点调试**
   - 在代码中设置断点
   - 在新窗口中操作
   - 原窗口会停在断点处

3. **热重载**
   - 修改代码后，在调试控制台点击"重启"按钮
   - 或停止调试（Shift+F5）后重新启动（F5）
